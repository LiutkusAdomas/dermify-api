## Tech Debt

### 1. Application-Level Editability Checks on Update Paths (Advisory)

Module and outcome update methods do not check session editability at the application layer:
- `EnergyModuleService`: UpdateIPLModule, UpdateNdYAGModule, UpdateCO2Module, UpdateRFModule
- `InjectableModuleService`: UpdateFillerModule, UpdateBotulinumModule
- `OutcomeService`: UpdateOutcome

DB immutability triggers prevent data corruption, but the API returns a database exception (500) instead of a clean 409 for update attempts on signed/locked sessions.

**Impact:** Error response quality degradation for edge cases only. Data integrity is preserved.

### 2. SUMMARY.md Frontmatter Metadata (Info)

None of the 23 plan SUMMARY.md files populate `requirements_completed`. Documentation metadata gap only.
