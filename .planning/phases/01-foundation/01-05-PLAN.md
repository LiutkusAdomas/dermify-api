Rewrite internal/api/handlers/patients_test.go to remove the //go:build ignore tag and implement 3 real test functions. The tests should follow the established pattern from internal/api/middleware/auth_test.go but test at the handler level through a chi router.

Test setup pattern for each test:
1. Create a test JWT secret and config via config.Configuration{Auth: config.AuthConfig{JWTSecret: secret, AccessTokenExpiry: 1*time.Hour}}.
2. Create a MockPatientRepository from testutil with a ListFn that returns an empty result (the point is testing RBAC, not patient logic).
3. Create a PatientService from the mock repository.
4. Create a metrics.Client via metrics.New(slog.Default()).
5. Build a chi.NewRouter() with the middleware chain: middleware.RequireAuth(cfg), middleware.RequireRole(allowed...), then mount the handler.
6. Generate a JWT token with the test role using auth.GenerateAccessToken.
7. Create httptest.NewRequest with Authorization: Bearer {token}, use httptest.NewRecorder.
8. Assert the expected HTTP status code and, for error cases, decode the apierrors.ErrorResponse and check the Code field.

Three test functions to implement:

**TestDoctorAccess_PatientsEndpoint (RBAC-02):**
- Generate a token with role="doctor".
- Mount HandleListPatients on GET / behind RequireAuth + RequireRole(domain.RoleDoctor, domain.RoleAdmin).
- Send GET request with the doctor token.
- Assert HTTP 200 response (doctor has access to patient endpoints).

**TestAdminAccess_PatientsEndpoint (RBAC-03):**
- Generate a token with role="admin".
- Same router setup as above.
- Send GET request with the admin token.
- Assert HTTP 200 response (admin can manage patients).

**TestUnauthorizedAccess_PatientsEndpoint:**
- Generate a token with role="" (empty string, simulating unassigned user).
- Same router setup as above.
- Send GET request with the no-role token.
- Assert HTTP 403 response.
- Decode the error body and assert Code == apierrors.AuthInsufficientRole.

Keep the file in package handlers_test (external test package). Import: testing, net/http, net/http/httptest, encoding/json, log/slog, time, config, auth, middleware, handlers, metrics, domain, service, testutil, testify/assert, testify/require.

Use a helper function (e.g., newPatientTestRouter) to reduce duplication across the 3 tests. The helper should accept the allowed roles for RequireRole and return the chi.Router.

Ensure all comments end with periods (godot linter). Do not use package-level variables (gochecknoglobals). Keep functions under 100 lines (funlen).
