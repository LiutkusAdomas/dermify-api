Create `internal/service/energy_module.go` containing:

1. **Repository interfaces** (4 total) -- one per module type, each with three methods:
   - `Create(ctx context.Context, detail *domain.XxxModuleDetail) error`
   - `GetByModuleID(ctx context.Context, moduleID int64) (*domain.XxxModuleDetail, error)`
   - `Update(ctx context.Context, detail *domain.XxxModuleDetail) error`
   Interface names: IPLModuleRepository, NdYAGModuleRepository, CO2ModuleRepository, RFModuleRepository

2. **Sentinel errors** (using `errors.New` with `//nolint:gochecknoglobals` comment, matching project convention):
   - `ErrModuleDetailNotFound` -- detail row not found
   - `ErrModuleDetailVersionConflict` -- optimistic locking failure on detail update
   - `ErrDeviceTypeMismatch` -- device type doesn't match module type (e.g., IPL module referencing RF device)
   - `ErrHandpieceMismatch` -- handpiece doesn't belong to the specified device
   - `ErrInvalidModuleData` -- generic validation failure on module detail fields

3. **EnergyModuleService struct** with fields:
   - sessionSvc *SessionService (for AddModule which enforces consent gate + editability)
   - registrySvc *RegistryService (for device/handpiece validation)
   - iplRepo IPLModuleRepository
   - ndyagRepo NdYAGModuleRepository
   - co2Repo CO2ModuleRepository
   - rfRepo RFModuleRepository

4. **Constructor:** `NewEnergyModuleService(sessionSvc *SessionService, registrySvc *RegistryService, iplRepo IPLModuleRepository, ndyagRepo NdYAGModuleRepository, co2Repo CO2ModuleRepository, rfRepo RFModuleRepository) *EnergyModuleService`

5. **Private helper:** `validateDeviceForModule(ctx context.Context, deviceID int64, handpieceID *int64, expectedDeviceType string) error` -- validates device exists via registrySvc.GetDeviceByID, checks device.DeviceType == expectedDeviceType, and if handpieceID is provided validates it belongs to the device by iterating device.Handpieces.

6. **Create methods** (4 total): `CreateIPLModule`, `CreateNdYAGModule`, `CreateCO2Module`, `CreateRFModule`. Each:
   - Takes (ctx, sessionID int64, detail *domain.XxxModuleDetail, userID int64) returns (*domain.XxxModuleDetail, error)
   - Calls validateDeviceForModule with the appropriate device type string (domain.ModuleTypeIPL, etc.)
   - Calls s.sessionSvc.AddModule(ctx, sessionID, domain.ModuleTypeXxx, userID) to create the base session_modules row (this enforces consent gate, editability, and sets sort order)
   - Sets detail.ModuleID = baseModule.ID, detail.CreatedBy/UpdatedBy = userID, detail.Version = 1
   - Calls the appropriate repo.Create to insert the detail row
   - Returns the detail

7. **Get methods** (4 total): `GetIPLModule`, `GetNdYAGModule`, `GetCO2Module`, `GetRFModule`. Each:
   - Takes (ctx, moduleID int64) returns (*domain.XxxModuleDetail, error)
   - Calls the appropriate repo.GetByModuleID

8. **Update methods** (4 total): `UpdateIPLModule`, `UpdateNdYAGModule`, `UpdateCO2Module`, `UpdateRFModule`. Each:
   - Takes (ctx, detail *domain.XxxModuleDetail, userID int64) returns error
   - Sets detail.UpdatedBy = userID
   - Calls validateDeviceForModule if DeviceID is set (allows changing device on update)
   - Calls the appropriate repo.Update
