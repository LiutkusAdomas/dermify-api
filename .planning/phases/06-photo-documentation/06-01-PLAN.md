   - Save(ctx context.Context, relPath string, reader io.Reader) (int64, error)
   - Delete(ctx context.Context, relPath string) error
   - Exists(ctx context.Context, relPath string) (bool, error)

2. PhotoRepository interface:
   - Create(ctx context.Context, photo *domain.Photo) error
   - GetByID(ctx context.Context, id int64) (*domain.Photo, error)
   - ListBySession(ctx context.Context, sessionID int64) ([]*domain.Photo, error)
   - ListByModule(ctx context.Context, moduleID int64) ([]*domain.Photo, error)
   - Delete(ctx context.Context, id int64) error

3. Sentinel errors:
   - ErrPhotoNotFound = errors.New("photo not found")
   - ErrPhotoConsentRequired = errors.New("photo consent required")
   - ErrPhotoInvalidType = errors.New("invalid photo type")
   - ErrPhotoSessionNotEditable = errors.New("session not editable for photo operations")
   - ErrPhotoModuleRequired = errors.New("module ID required for label photos")
   - ErrPhotoInvalidContentType = errors.New("invalid content type")

4. PhotoService struct with fields:
   - repo PhotoRepository
   - sessionRepo SessionRepository (reuse existing interface from session.go)
   - moduleRepo ModuleRepository (reuse existing interface from session.go for module existence check)
   - fileStore FileStore

5. NewPhotoService(repo PhotoRepository, sessionRepo SessionRepository, moduleRepo ModuleRepository, fileStore FileStore) *PhotoService

6. UploadPhoto(ctx context.Context, photo *domain.Photo, reader io.Reader) error:
   - Fetch session via sessionRepo.GetByID (return wrapped ErrSessionNotFound on failure)
   - Check consent gate: if session.PhotoConsent is nil or *session.PhotoConsent == domain.PhotoConsentNo, return ErrPhotoConsentRequired
   - Check session is editable: status must be draft or in_progress (return ErrPhotoSessionNotEditable otherwise)
   - Validate photo_type is "before" or "label" (return ErrPhotoInvalidType)
   - If photo_type is "label", validate ModuleID is not nil (return ErrPhotoModuleRequired)
   - Validate content type is in domain.AllowedPhotoContentTypes (return ErrPhotoInvalidContentType)
   - Generate file path: sessions/{session_id}/{photo_type}/{hex_id}.{ext}
     - Use crypto/rand for 16 random bytes, hex encode for filename
     - Map content type to extension: image/jpeg -> .jpg, image/png -> .png
   - Set photo.FilePath to the generated relative path (use forward slashes via path.Join, not filepath.Join, for storage)
   - Save file via fileStore.Save (get written bytes)
   - Set photo.SizeBytes from fileStore.Save return value if not already set
   - Create metadata via repo.Create
   - If repo.Create fails, attempt fileStore.Delete to clean up orphaned file (log error but don't mask original)
   - Return nil on success

7. GetByID(ctx context.Context, id int64) (*domain.Photo, error)

8. ListBySession(ctx context.Context, sessionID int64) ([]*domain.Photo, error)

9. ListByModule(ctx context.Context, moduleID int64) ([]*domain.Photo, error)

10. DeletePhoto(ctx context.Context, id int64, userID int64) error:
    - Fetch photo via repo.GetByID
    - Fetch session via sessionRepo.GetByID
    - Check session is editable (draft or in_progress)
    - Delete file via fileStore.Delete
    - Delete metadata via repo.Delete
    - Return nil on success

Use a helper function isSessionEditable(status string) bool that checks for draft or in_progress, matching the existing isEditable pattern in session service. Use strconv and path (not path/filepath) for the relative path generation to keep POSIX-style forward slashes.
